The one-line version

It is not “an AI translator.” It’s a governed review workflow that gets smarter every time our team uses it. A machine writes a first draft; our people review and correct it; and every correction — the right term, the preferred phrasing, the regionalism to avoid — becomes reusable, auditable institutional memory applied automatically to the next document. The asset isn’t the translation; it’s the memory and the audit trail.

How the memory is invoked

The team’s approved terms, phrasings and “avoid this regionalism” rules aren’t applied as a clean-up at the end — they are loaded first and drive the whole process.

01 · LOADED UP FRONT

Glossary, active rules and prior translations load before anything is translated.

02 · PRIMES THE DRAFT

Fed into the translator’s instructions, so the first draft already uses our wording.

03 · ENFORCED THROUGHOUT

Re-applied as hard constraints on every rewrite; the checks flag any drift.

04 · FEEDS BACK IN

A reviewer’s correction becomes a governed rule, applied to every future document.

The cross-model cycle, step by step

A single model can’t reliably catch its own mistakes. So a different AI independently reviews the first one’s work — twice — and the document is only rewritten where it objectively needs it.

01 · DRAFT
Sonnet writes
Claude Sonnet 4.6 drafts the Spanish, primed with our glossary & rules
02 · SCORE + CRITIQUE
QE scores, GPT-5 critiques
our Quality-Estimation model scores 0–1 to route effort (never approves); GPT-5, a different AI, returns the exact problems
03 · REFINE
Sonnet refines using GPT-5’s notes
rewrites only the flagged spans; our rules re-applied as hard constraints
04 · RE-SCORE + RE-CRITIQUE
QE re-scores, GPT-5 critiques again
confirms the fix landed and introduced nothing new
05 · KEEP THE BEST
keep if better, else revert
kept only if QE and the critic agree it improved; repeats until clean (hard cap); good text is left untouched
In short: Sonnet writes → QE scores + GPT-5 critiques → Sonnet refines using GPT-5’s notes → QE re-scores + GPT-5 critiques again → keep the best. Quality can only stay the same or get better — never worse.

What’s deterministic vs. what’s AI

This is the part that makes it trustworthy in a regulated setting.

AI · judgment
  • The first-draft translation
  • The independent cross-model review
  • The QE routing score (where to focus effort)
Deterministic · rules, no AI
  • Number & currency checks (English “billion” never becomes Spanish “billón”)
  • Glossary & regionalism enforcement
  • The rule that AI can never approve
  • An append-only audit log of every change

The final Spanish is decided by the deterministic checks plus a human sign-off — never by an AI’s confidence score.

The models we call

RoleModelWhy
TranslatorClaude Sonnet 4.6Register & tone consistency over long financial prose. Also does the targeted rewrites.
CriticGPT-5Deliberately a different family from the translator, so it doesn’t share its blind spots.
Quality Estimation (QE)Self-hostedA small model on our own server that scores each segment 0–1 to route effort — no external service, no per-call cost. Routing signal only; it never approves.

All three are configuration, not code — we can swap providers or models by editing one file, no rebuild.
