What it really is

Most "AI translation" hands you a draft and stops there. This keeps a person in charge and remembers what they fix. Every correction (a preferred term, a regionalism we want to avoid) becomes a governed rule and gets reused on the next document. The thing we're actually building isn't the translation. It's the memory and the audit trail behind it.

How the memory is invoked

The team’s approved terms, phrasings and “avoid this regionalism” rules aren’t a clean-up at the end. They’re loaded first and drive the whole process.

01 · LOADED UP FRONT

Glossary, active rules and prior translations load before anything is translated.

02 · PRIMES THE DRAFT

Fed into the translator’s instructions, so the first draft already uses our wording.

03 · ENFORCED THROUGHOUT

Re-applied as hard constraints on every rewrite; the checks flag any drift.

04 · FEEDS BACK IN

A reviewer’s correction becomes a governed rule, applied to every future document.

There’s also a separate Train page: paste in a document we’ve already translated (English left, finished Spanish right) and it captures those pairings straight into the memory. So we can seed it from work the team has already done, not only from corrections made inside the tool.

The cross-model cycle, step by step

A model can’t reliably catch its own mistakes. So a different AI reviews the first one’s work, twice, and the document only gets rewritten where it genuinely needs it.

01 · DRAFT
Sonnet writes
Claude Sonnet 4.6 drafts the Spanish, primed with our glossary & rules
02 · SCORE + CRITIQUE
QE scores, GPT-5 critiques
our Quality-Estimation model scores 0–1 to route effort (never approves); GPT-5, a different AI, returns the exact problems
03 · REFINE
Sonnet refines using GPT-5’s notes
rewrites the flagged spans (told to leave good text alone); our rules re-applied as hard constraints
04 · RE-SCORE + RE-CRITIQUE
QE re-scores, GPT-5 critiques again
confirms the fix landed and introduced nothing new
05 · KEEP THE BEST
keep if better, else revert
kept only if it removes a serious (major/critical) problem, or scores higher without adding one; otherwise we throw it away. Repeats until clean, with a hard cap. Good text is left alone.
In short: Sonnet writes → QE scores + GPT-5 critiques → Sonnet refines using GPT-5’s notes → QE re-scores + GPT-5 critiques again → keep the best. Because a worse rewrite gets discarded, the text only ever stays the same or gets better.

What’s deterministic vs. what’s AI

This is the part that makes it trustworthy in a regulated setting.

AI · judgment
  • The first-draft translation
  • The independent cross-model review
  • The QE routing score (where to focus effort)
Deterministic · rules, no AI
  • Number & currency checks (English “billion” never becomes Spanish “billón”)
  • Glossary & regionalism enforcement
  • The rule that QE/AI can never approve; validators and a human decide
  • A full edit and hand-off log: every change is appended as an event, never overwritten

The final Spanish is decided by the deterministic checks and a human sign-off, never by an AI’s confidence score.

The models we call

RoleModelWhy
TranslatorClaude Sonnet 4.6Register & tone consistency over long financial prose. Also does the targeted rewrites.
CriticGPT-5Deliberately a different family from the translator, so it doesn’t share its blind spots.
Quality Estimation (QE)Self-hostedA small model on our own server that scores each segment 0–1 to route effort. No external service, no per-call cost. Routing signal only; it never approves.

Model choice and settings are configuration, not code. We can change the model or tune it by editing one file, no rebuild.
